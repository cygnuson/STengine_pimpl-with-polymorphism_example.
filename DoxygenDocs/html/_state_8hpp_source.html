16 /**********************The state system.***************************************
17 * A: The app keeps a stack of states. The stack cannot be alterd by a state.
18 * B: The State's function `State:Flag HandleInput(sf::Event&,float)` can return
19 * a flag that will ask the app to push a new state to the stack, or pop one.
20 * remember that if the pop signal is returned from HandleInput, the state
21 * that returned the flag will be popped.
22 * C:When the HandleInput function rturns the `Push` flag, or any flag that
23 * requires a new state, the app will call `std::shared_ptr<State> GetState()`
24 * to get the state that is needed to be pushed. The reason is that the
25 * alternative is returning a pair on every call (thousands of calls), thats
26 * not a good idea as it would hinder performance.
27 * D:The ONLY acceptable way to make a state for the system is to call
28 * `State::MakeState<SomeState>(SomeState's args)`. SomeState is not required
29 * to inherit anything. but it MUST have the following functions defined:
30 *
31 * bool Draw(sf::RenderWindow&, sf::Time)
32 * State::Flag HandleInput(sf::Event&, sf::Time)
33 * sf::View& GetView()
34 * bool SanityCheck()
35 * bool UpdateLogic(sf::Time)
36 * std::shared_ptr<State> GetState()
37 * void Freeze();
38 * void Unfreeze();
39 *
40 * This list may change, its also at the function comment for
41 * MakeState<..>(...)
42 *
43 * E:The State class is just a wrapper that pipes all calls to the impl in _self
44 * which is a pointer to a StateBase which is just a virtual wrapper for
45 * Model<Type> so that it can be stored unirsally in the _self ptr without
46 * having a million different State types to keep track of. This means that
47 * ever call on the state from outside that state will have one regular funct
48 * call and one additional virtual call. This means that its best to call from
49 * inside the state as often as able.
50 *
51 * F: The state implamentor is responsible for timing. All that is supplied
52 * is the delta-time as a function arg.
53 *
54 * G:The function GetState() should only allow a state to be accessed ONCE! The
55 * best way to do it is to set it to null after giving someone the state.
56 *
57 * H:The Freeze() and Unfreeze() functions are executed when the state
58 * changes(freeze), and changes back(unfreeze).
59 *
60 *
61 ******************************************************************************/
